# Requirements

Use stable `REQ-x.y` identifiers so requirements can be cited without repeating their meaning. Keep each requirement atomic, externally observable, and testable. An accepted requirement that is ready for implementation names its applicable interface or boundary and a falsifiable verification path. If automation would not be meaningful, record a concrete manual check.
